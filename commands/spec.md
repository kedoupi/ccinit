# Spec 命令

规格驱动开发模式，从需求到实施的完整文档生成：

**工作流程**：
1. **需求澄清** - 用户故事、业务规则、约束条件
2. **设计探索** - 架构选型、数据模型、接口定义
3. **实施规划** - 任务拆解、优先级、测试策略

**输出文档**：
- `requirements.md` - 功能需求和边界条件
- `design.md` - 系统架构和技术方案
- `tasks.md` - 实施计划和验收标准

每个阶段都会要求确认后再继续，确保文档符合期望。适合新功能开发、系统改造等需要完整规划的场景。
